[문제 8393](https://www.acmicpc.net/problem/8393)

# 합

## 문제

n이 주어졌을 때, 1부터 n까지 합을 구하는 프로그램을 작성하시오.

## 입력

첫째 줄에 n (1 ≤ n ≤ 10,000)이 주어진다.

## 출력

1부터 n까지 합을 출력한다.

## 예제 입력

```
3
```

## 예제 출력

```
6
```

---

## ✏️ 문제 풀이

```
const fs = require('fs');
const input = fs.readFileSync('/dev/stdin').toString()
```

```
const num = Number(input)
let sumNum = 0

for (let i = 0; i <= num; i++) {
  sumNum += i
}

console.log(sumNum)
```

## 💡 추가 풀이

### 1. 삼각형 넓이 공식으로 구현 `밑변 X 높이 / 2`

#### 설명

`n = 5`일 경우 삼각형을 그림으로 표현해보면 아래와 같다.  
 `5 X 5 / 2` 이 공식으로 하면 원하는 값이 나오지 않는다.

```
◯
◯ ◯
◯ ◯ ◯
◯ ◯ ◯ ◯
◯ ◯ ◯ ◯ ◯
```

구하고자하는 **삼각형의 2배**를 가진 사각형을 만들면 아래와 같다.

```
◯ ● ● ● ● ●
◯ ◯ ● ● ● ●
◯ ◯ ◯ ● ● ●
◯ ◯ ◯ ◯ ● ●
◯ ◯ ◯ ◯ ◯ ●
```

`5 X (5 + 1) / 2` → `n X (n + 1) / 2`

이러한 방법을 수학적인 사고 방법이라고 한다. 수학은 복잡한 것을 단순하게 만들어 놓고 계산한다. 이 것이 바로 **추상화**의 과정이다.

누적 계산 문제를 반복적으로 풀었을 때는 이해가 쉽지만 n이 무한히 커지면 수행 속도가 저하된다.  
반면 수식으로 문제를 풀었을 때는 성능에는 탁월하지만 공식으로 푼 코드만 보고는 어떠한 일을 하는지 파악하기 힘들 수 있다.  
[참고링크](https://adgw.tistory.com/entry/%EC%95%8C%EA%B3%A0%EB%A6%AC%EC%A6%98-1-n%EA%B9%8C%EC%A7%80-%ED%95%A9%EC%9D%84-%EA%B5%AC%ED%95%98%EB%8A%94-%EC%9B%90%EB%A6%AC)

#### 코드

```
const num = Number(input)
const sumNum = num * (num + 1) / 2

console.log(sumNum)
```
